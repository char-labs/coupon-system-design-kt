# Harness 현황 조회

이 프로젝트에 구성된 에이전트 팀과 스킬을 읽어 현황을 출력한다.

## 실행 절차

### Step 1: 에이전트 목록 수집

`.claude/agents/` 디렉토리의 모든 `.md` 파일을 읽는다. 각 파일에서 다음을 추출한다:
- **핵심 역할** (첫 번째 섹션)
- **빌트인 타입** (general-purpose / Explore / Plan)
- **소속 팀** (어떤 오케스트레이터가 참조하는지)

### Step 2: 스킬 목록 수집

`.claude/skills/` 디렉토리의 각 `SKILL.md` 파일에서 frontmatter의 `name`과 `description`을 읽는다. 오케스트레이터와 전문 스킬을 구분한다:
- **오케스트레이터**: description에 "팀을 조율", "orchestrator" 포함
- **전문 스킬**: 에이전트가 사용하는 개별 스킬

### Step 3: 팀 구조 파악

오케스트레이터 스킬을 기준으로 어떤 에이전트들이 한 팀인지 매핑한다.

### Step 4: 현황 출력

아래 형식으로 출력한다:

```
# 🏗️ Harness 현황 — {프로젝트명}

## 팀 목록

### 팀 1: {팀명}
- **오케스트레이터 스킬:** `{skill-name}`
- **트리거:** "{트리거 키워드 예시}"
- **팀원:**
  | 에이전트 | 타입 | 역할 요약 | 전담 스킬 |
  |---------|------|---------|---------|
  | ... | ... | ... | ... |
- **실행 패턴:** {팬아웃/팬인 / 파이프라인 / 감독자}
- **산출물:** {최종 출력물}

---

## 단독 스킬 (팀 없이 직접 사용)

| 스킬 | 트리거 | 설명 |
|------|--------|------|
| ... | ... | ... |

---

## 에이전트 전체 목록

| 에이전트 | 타입 | 소속 팀 |
|---------|------|--------|
| ... | ... | ... |

---

## 사용 방법

### 팀 실행
{오케스트레이터 트리거 예시를 팀별로 나열}

### 단일 에이전트 작업
{에이전트 직접 호출이 필요한 경우 예시}
```

## 출력 원칙

- 파일을 읽을 때 모든 에이전트와 스킬을 빠짐없이 포함한다
- 트리거 문구는 description에서 가장 자연스러운 예시를 1~2개 추출한다
- 팀원 역할 요약은 한 줄로 간결하게 작성한다
- 새 팀이나 스킬을 추가하라는 요청이 나와도 이 스킬에서는 응답하지 않는다 — 그 경우 `harness:harness` 마켓플레이스 스킬을 사용하라고 안내한다
