# 스킬 & 에이전트 사용 가이드

이 프로젝트에서 스킬과 에이전트를 어떻게 쓰는지 설명한다.

## 실행 절차

아래 내용을 사용자에게 그대로 출력한다.

---

## 기본 개념

**스킬(Skill)** 은 Claude에게 "이런 요청이 오면 이렇게 행동하라"고 알려주는 지침서다.
평소에 Claude에게 말하듯이 자연어로 요청하면, 관련 스킬이 자동으로 적용된다.

**에이전트(Agent)** 는 특정 역할을 맡은 Claude 인스턴스다.
에이전트 팀이 실행되면 여러 에이전트가 동시에 작업하고 서로 소통하며 결과를 만든다.

---

## 사용 방법 — 그냥 말하면 된다

스킬을 직접 호출하거나 에이전트를 직접 실행할 필요가 없다.
**자연어로 요청하면 Claude가 알아서 적절한 스킬을 선택해 실행한다.**

### 예시

```
"코드 리뷰해줘"
→ code-review-orchestrator 스킬 실행
→ 아키텍처/보안/성능/스타일 4명이 병렬 감사 시작
```

```
"Kafka 도입해서 10k RPS 달성하고 싶어"
→ highscale-coupon-orchestrator 스킬 실행
→ 설계 → 구현 → 테스트 → 모니터링 순서로 팀 작업
```

```
"팀 현황 보여줘"
→ team-status 스킬 실행
→ 현재 구성된 에이전트와 스킬 목록 출력
```

## 에이전트 팀이 실행되면 어떻게 되나?

오케스트레이터 스킬을 요청하면 아래 흐름이 자동으로 진행된다:

```
1. Claude가 _workspace/ 폴더 생성
2. 팀원 에이전트들 병렬 실행 시작
3. 에이전트들이 코드를 읽고 작업 수행
4. 필요하면 에이전트끼리 메시지 주고받으며 조율
5. 각자 결과 파일 저장 (_workspace/01_..., 02_... 등)
6. Claude가 결과 통합해서 최종 리포트 생성
7. 사용자에게 요약 보고
```

중간에 별도로 할 일은 없다. Claude가 알아서 진행하고 완료되면 결과를 보고한다.

## 팁

**범위를 좁혀서 요청할 수 있다:**
```
"coupon-domain만 코드 리뷰해줘"
"Kafka 말고 RabbitMQ로 도입해줘"
"성능 감사만 먼저 해줘"
```

**새 팀이나 스킬을 추가하고 싶을 때:**
```
"새 하네스 구성해줘 — OO 기능을 자동화하고 싶어"
→ harness:harness 마켓플레이스 스킬이 실행되어 설계부터 도와준다
```

**결과 파일 위치:**
- 중간 산출물: `_workspace/` 폴더 안
- 최종 리포트: 각 오케스트레이터 스킬에 명시된 경로
